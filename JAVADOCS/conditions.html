Question 1:

if (5 > 10) {
   console.log('5 is greater than 10');
} else {
   console.log('5 is less than or equal to 10');
}
Output: 5 is less than or equal to 10
Explanation: The condition 5 > 10 is false, so the else block is executed.

Question 2:

if (NaN === NaN) {
   console.log('NaN is equal to NaN');
} else {
   console.log('NaN is not equal to NaN');
}
Output: NaN is not equal to NaN
Explanation: In JavaScript, NaN is not equal to NaN. This is due to the IEEE 754 standard for floating point numbers.

Question 3:

if (!null) {
   console.log('null is truthy');
} else {
   console.log('null is falsy');
}
Output: null is falsy
Explanation: null is considered falsy in JavaScript. The expression !null evaluates to true.

Question 4:

if (0 === '') {
   console.log('0 is equal to empty string');
} else {
   console.log('0 is not equal to empty string');
}
Output: 0 is not equal to empty string
Explanation: 0 (number) and '' (empty string) are of different types and thus not equal in JavaScript.

Question 5:

if (true && false) {
   console.log('true and false is true');
} else {
   console.log('true and false is false');
}
Output: true and false is false
Explanation: The expression true && false results in false because both operands must be true for the && (AND) operator to return true.

Question 6:

if (false || true) {
   console.log('false or true is true');
} else {
   console.log('false or true is false');
}
Output: false or true is true
Explanation: The expression false || true results in true because the || (OR) operator returns true if at least one operand is true.

Question 7:

if (!false && true) {
   console.log('not false and true is true');
} else {
   console.log('not false and true is false');
}
Output: not false and true is true
Explanation: !false evaluates to true (since ! negates the falsy value false), and true && true results in true.

Question 8:

if (5 === '5') {
   console.log('5 is equal to string 5');
} else {
   console.log('5 is not equal to string 5');
}
Output: 5 is not equal to string 5
Explanation: 5 (number) and '5' (string) are of different types in JavaScript, so they are not equal.

Question 9:

if (undefined == null) {
   console.log('undefined is equal to null');
} else {
   console.log('undefined is not equal to null');
}
Output: undefined is equal to null
Explanation: undefined loosely equals null in JavaScript (using ==), although their types are different.

Question 10:

if (typeof [] === 'object') {
   console.log('Array is an object');
} else {
   console.log('Array is not an object');
}
Output: Array is an object
Explanation: In JavaScript, arrays are of type object.
