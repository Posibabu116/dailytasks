JavaScript Variable Declarations

Differences between var, let, and const

Feature var let const
Scope Function Block Block
Hoisting Yes, initialized with undefined Yes, but not initialized Yes, but not initialized
Reassignment Yes Yes No
Redeclaration Yes No No

var

The var keyword is used to declare a variable in JavaScript. It has function scope, which means that the variable is only accessible within the function it is declared in. If declared outside any function, it has global scope.

var x = 10;
function testVar() {
    var y = 20;
    console.log(x); // 10
    console.log(y); // 20
}
testVar();
console.log(y); // ReferenceError: y is not defined

let

The let keyword is used to declare a variable with block scope. This means that the variable is only accessible within the block (e.g., a pair of curly braces) it is declared in.

let a = 30;
if (true) {
    let b = 40;
    console.log(a); // 30
    console.log(b); // 40
}
console.log(b); // ReferenceError: b is not defined

const

The const keyword is used to declare a constant variable with block scope. This variable cannot be reassigned, although the contents of objects and arrays can still be modified.

const c = 50;
if (true) {
    const d = 60;
    console.log(c); // 50
    console.log(d); // 60
}
console.log(d); // ReferenceError: d is not defined

// Const with objects
const obj = { name: "John" };
obj.name = "Jane"; // This is allowed
console.log(obj.name); // "Jane"

// Const with arrays
const arr = [1, 2, 3];
arr.push(4); // This is allowed
console.log(arr); // [1, 2, 3, 4]
